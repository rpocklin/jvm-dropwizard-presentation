    • From 50,000 feet

  • No Hibernate
  • No Spring
  • No DI
  • Jetty, Jersey, Jackson, Hibernate Validations
  • JDBI, Liquibase
  • Packaged as a single JAR

  • public static void main() {..}

  • Running it Locally

  • 5 - 10 seconds to startup
  • great support for CLI
  • Debugging is soo easy!

    • Testing it

  • JUnit, Rest Assured, Mockito
  • JSON Schema Validation
  • H2 Database

  • Deploying it

  • Make Jar not War
  • small JAR ( approx 27MB )
  • DB Migrations - Liquibase
  • To run:
  • java -jar app.jar server production.yml

  • Learnings

  • API and Workflow tests
  • Separate view and DB model POJOS
  • Create: re-read from DB may be required
  • Liquibase requires an artifact

  • Strengths

  • Productive and simple
  • Build it up as needed
  • Fast and performant
  • Good eco-system support
  • JDBI keeps SQL transparent
  • Great choice for small or micro-services

    • Weaknesses

  • Lack of batch inserts with JDBI
  • Multi-resource transaction scope is tricky
  • Enforces separation of Resources
  • so, need to consider urls as part of design
  • No OOTB dependency injection
  • so, less support from Camel / RabbitMQ
